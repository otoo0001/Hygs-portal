# HYGS Groundwater Portal

An interactive browser-based portal for exploring, visualising, and exporting global groundwater observation well data from the **geowat** PostgreSQL database (Utrecht University).

Built as part of PhD research on groundwater-dependent wetlands (wetGDEs) at global scale.

---

## What it does

- **Map view** — plots all wells as dots on an OpenStreetMap background. Colour by observation density or by long-term trend direction (declining / rising / flat)
- **Time series viewer** — click any well to load its monthly groundwater head, salinity, and extraction time series with an automatic linear trend overlay
- **Lithology profile** — shows the bore lithology depth profile (GLIM classes) for wells that have it
- **Spatial analysis** — draw a bounding box on the map to get the average time series and trend statistics for all wells in the selection
- **Filtering** — filter by country, region/state (with autocomplete), elevation, observation density, data flags (GWH / GWS / GWE / lithology), and bounding box
- **Export builder** — export well metadata, head time series, salinity time series, extraction time series, and lithology profiles as CSV, for any combination of scope (single well / country / bounding box / all filtered wells)

## Data sources

| Table | Variable | Unit |
|---|---|---|
| `_gwh_monthly_tb` | Groundwater head | m (converted from cm) |
| `_gws_monthly_tb` | Groundwater salinity | ppm |
| `_gwe_monthly_tb` | Groundwater extraction | m³/d |
| `_bore_litho_tb` | Bore lithology by depth interval | GLIM class ID |
| `_lookup_tb` | Well coordinates, flags, elevation | WGS84 / m asl |

Database: **geowat** @ ages-db01.geo.uu.nl (UU internal network / VPN required)
Period: 1979–2019

## Setup

### Requirements

```bash
pip install psycopg2-binary pandas geopandas requests
```

### Step 1 — Export data from the database (UU VPN required)

Run once to generate the static data files the portal reads:

```bash
python export_to_json.py
```

This produces a `data/` folder containing:

```
data/
├── wells.json               # all well metadata + pre-computed trend slopes
├── countries.json           # country list for filter dropdown
├── timeseries/{wellID}.json # monthly GWH series per well
├── salinity/{wellID}.json   # monthly salinity series per well
├── extraction/{wellID}.json # monthly extraction series per well
└── lithology/
    ├── litho_classes.json   # GLIM class lookup
    └── lithology_all.json   # all bore lithology profiles in one file
```

Re-run any time you want to refresh the data.

### Step 2 — Serve the portal locally

```bash
python -m http.server 8000
```

Then open your browser at `http://localhost:8000/portal.html`.

Or double-click `start_portal.bat` on Windows.

> **Note:** You must use a local server, not open `portal.html` directly by double-clicking, because browsers block `fetch()` calls on `file://` URLs.

## Deploying to a server (e.g. Snellius)

Copy the project to the server and serve it there:

```bash
scp -r portal.html data/ otoo0001@snellius.surf.nl:/gpfs/scratch1/shared/otoo0001/gw-portal/
ssh otoo0001@snellius.surf.nl
cd /gpfs/scratch1/shared/otoo0001/gw-portal
python -m http.server 8000
```

Then on your laptop open an SSH tunnel:

```bash
ssh -L 8000:localhost:8000 otoo0001@snellius.surf.nl
```

And open `http://localhost:8000/portal.html` in your browser.

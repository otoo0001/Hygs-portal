Data coverage & limitations

Coverage summary

Temporal coverage Export covers 1979–2019 from _gwh_monthly_tb. num_months reflects actual non-null observations, not potential.
Units Raw DINO values in cm, divided by 100 at export. All gwh_m values are in metres relative to local datum.
Data flags litho, gwh, gws, gwe are boolean columns in _lookup_tb. They indicate whether that variable type is available for a well, not the values themselves.
Elevation glo90_elev_m_asl is from the GLO-90 DEM. orig_elev_m_asl is from the original data source. Both may be null for some wells.
Coordinates WGS84 (EPSG:4326). x_wgs84 stored as lon, y_wgs84 stored as lat in the exported JSON.
Refreshing the dataset Re-run export_to_json.py on the UU network (VPN or campus) to pull updated records from geowat.

Known issues

Outlier head values Some wells contain erroneous spikes. Apply a plausibility check (e.g. exclude values below -30 m or above 50 m) before trend analysis.
Null elevation Some wells have no elevation value in either elevation column. These appear as — in the table and are excluded from elevation range filters.
